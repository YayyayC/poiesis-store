# POIESIS 배포 가이드 (실제 인터넷에 올리기)

지금 상태:
- **백엔드(poiesis-server)** + **프런트엔드(poiesis)** 가 하나로 동작합니다. 서버가 프런트까지 함께 서빙해요.
- 회원·세션·한정재고·주문·에디션 넘버는 **실제로 작동**(테스트로 검증됨).
- 카카오 로그인·토스 카드결제는 **코드 완성** 상태 — 당신의 키만 넣으면 켜집니다.

아래 순서대로 하면 인터넷에서 실제로 열립니다. (개발 경험 없어도 따라올 수 있게 적었어요.)

---

## A. 먼저 그대로 띄워보기 (결제·카카오 없이도 작동)
이 상태만으로도 회원가입·로그인·장바구니·한정재고·주문이 진짜로 돌아갑니다.
(카드는 "주문 접수"까지, 실결제는 C단계에서 활성화)

### 1) 두 폴더를 같은 위치에 둡니다
```
어떤폴더/
├─ poiesis/          (프런트엔드)
└─ poiesis-server/   (백엔드 — 이 폴더를 배포)
```

### 2) Render.com 으로 배포 (무료 플랜 가능)
1. github.com 에 두 폴더를 한 저장소(repo)로 올립니다. (git 사용)
2. render.com 가입 → **New → Web Service** → 해당 저장소 선택
3. 설정:
   - **Root Directory**: `poiesis-server`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Environment**: `Node`
4. **Environment** 탭에 환경변수 입력 (`.env.example` 참고):
   - `SESSION_SECRET` = 길고 무작위한 문자열 (필수)
   - `NODE_ENV` = `production`
   - `APP_ORIGIN` = Render가 준 주소 (예: `https://poiesis.onrender.com`)
5. **영구 디스크**(Disk) 추가 → 마운트 경로 `/data` → 환경변수 `DB_PATH=/data/poiesis.db`
   (이걸 안 하면 재배포 시 회원·주문 데이터가 사라집니다)
6. Deploy → 몇 분 뒤 `https://...onrender.com` 으로 사이트가 열립니다. 🎉

> Railway, Fly.io, Cloudtype(국내) 등도 동일한 방식으로 가능합니다.

---

## B. 도메인 + HTTPS 연결
1. 가비아·후이즈·Cloudflare 등에서 도메인 구입 (예: poiesis.co.kr)
2. Render의 **Custom Domain**에 도메인 추가 → 안내되는 DNS 레코드를 도메인 업체에 입력
3. HTTPS 인증서는 보통 자동 발급됩니다.
4. `APP_ORIGIN` 환경변수를 새 도메인으로 변경.

---

## C. 카카오 로그인 켜기
1. developers.kakao.com → 내 애플리케이션 → **애플리케이션 추가**
2. **앱 키 → REST API 키** 복사 → 환경변수 `KAKAO_CLIENT_ID`
3. **카카오 로그인** 활성화 ON → **Redirect URI** 등록:
   `https://your-domain.com/api/auth/kakao/callback`
   → 같은 값을 환경변수 `KAKAO_REDIRECT_URI` 에도 입력
4. **동의항목**에서 닉네임(필수)·이메일(선택) 설정
5. 재배포 → 로그인 화면의 "카카오로 계속하기"가 **실제로** 작동합니다.

---

## D. 토스페이먼츠(카드 결제) 켜기
1. developers.tosspayments.com 가입 → **테스트 키** 즉시 발급
2. 환경변수에 입력:
   - `TOSS_CLIENT_KEY` = `test_ck_...`
   - `TOSS_SECRET_KEY` = `test_sk_...`
3. 재배포 → 카드 결제가 **토스 결제창**으로 연결됩니다. 테스트 카드로 전 과정 확인 가능.
4. **실제 돈 결제**를 받으려면:
   - 사업자등록 + 통신판매업 신고 완료
   - 토스페이먼츠에 **계약·심사** 신청 → 승인되면 **라이브 키**(`live_ck_`, `live_sk_`)로 교체

> 결제 흐름은 안전하게 설계됨: 금액·재고를 서버가 다시 검증하고, 결제 성공 후 재고가 마감됐으면 자동 취소(환불)합니다.

## E. 운영 점검 체크리스트
- [ ] `SESSION_SECRET` 를 추측 불가한 긴 값으로 설정
- [ ] 영구 디스크(DB) 또는 PostgreSQL 연결 (데이터 보존)
- [ ] 도메인 HTTPS 정상
- [ ] 카카오 Redirect URI 정확히 일치
- [ ] 토스 라이브 전환 전 테스트 결제로 전 과정 점검
- [ ] 판매자 정보(푸터)·약관·개인정보 실제 정보로 교체
- [ ] 정기 백업

## 규모가 커지면
- 내장 SQLite → **PostgreSQL** 전환(Neon·Supabase 무료 플랜). `db.js`의 쿼리는 거의 그대로 이식 가능.
- 세션 저장소를 메모리 → Redis 등으로(다중 서버 시).

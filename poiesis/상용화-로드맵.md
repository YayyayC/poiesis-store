# POIESIS 상용화 로드맵

> 현재 사이트는 **프런트엔드 데모**입니다. 디자인·화면 흐름·장바구니·회원/결제 UI는 모두 동작하지만,
> 회원·주문·결제 데이터는 브라우저(localStorage)에만 저장됩니다. 실제 판매를 위해서는 아래 절차가 필요합니다.
>
> 우선순위 표기: 🔴 필수(없으면 판매 불가/불법) · 🟡 중요 · 🟢 권장

## 한눈에 보는 순서

1. 사업자 등록 → 2. 통신판매업 신고 → 3. 결제대행(PG) 계약 → 4. 백엔드/DB 구축(또는 솔루션 이전)
→ 5. 도메인·호스팅·SSL → 6. 법적 페이지 게시 → 7. 배송/CS 운영체계 → 8. QA·런칭

가장 빠른 길과 가장 자유로운 길 중 하나를 4번에서 선택하게 됩니다(아래 "기술 경로 선택" 참고).

## 2. 결제 (Payments) 🔴

실제 카드 결제는 **PG(결제대행사)** 계약 + **백엔드 결제 검증**이 있어야 합니다. 현재 결제 화면은 UI 데모입니다.

대표 PG / 간편결제
- **포트원(구 아임포트)** — 여러 PG·간편결제를 한 번에 연동. 개발 친화적, 스타트업에 인기
- **토스페이먼츠** — 문서·개발자 경험 우수, 토스/카카오/네이버페이 연동
- **KG이니시스 / 나이스페이먼츠** — 전통적 PG, 안정적
- 간편결제 직접 연동: **카카오페이, 네이버페이, 토스페이**

연동 시 필요한 백엔드 작업
- [ ] 결제창 호출 → 승인 → **서버에서 결제 금액·상태 재검증**(위변조 방지)
- [ ] 결제 완료 웹훅(Webhook) 수신 및 주문 상태 갱신
- [ ] 환불/부분취소 API 연동
- [ ] 정기 정산·세금계산서 처리

> ⚠️ 카드번호 등 민감정보는 직접 저장하지 마세요. PG가 PCI-DSS를 대신 처리합니다.

## 3. 기술 경로 선택 (Build vs. Platform) 🟡

상용화의 핵심 분기입니다. 둘 중 하나를 고르세요.

### 경로 A — 솔루션 이전 (빠르고 안전, 가장 추천)
**카페24 · 아임웹 · 식스샵 · Shopify** 같은 커머스 솔루션으로 옮기는 방법.
- 장점: 결제·배송·재고·회원·정산·보안이 **내장**. 운영자가 코딩 없이 관리. 최단 런칭.
- 작업: 현재 디자인(브랜드 톤/컬러/카피)을 솔루션 테마로 이식. 상품 등록, 결제·배송 설정.
- 적합: "지금 코드를 처음부터 백엔드까지 만들 여력이 없다 / 빨리 팔고 싶다"

### 경로 B — 자체 구축 (자유롭지만 개발 필요)
현재 프런트엔드를 유지하고 **백엔드를 직접 개발**.
- 필요한 백엔드: 회원 인증, 상품/재고 DB, 주문, 결제 연동, 관리자(어드민) 페이지, 이메일 발송
- 스택 예: Next.js(또는 현재 정적 + API 서버) + DB(PostgreSQL/MySQL) + 포트원/토스 + 어드민
- 장점: 디자인·기능 100% 자유. 장점이자 비용.
- 적합: "브랜드 경험을 완전히 통제하고 싶다 / 개발 리소스가 있다"

체크리스트(경로 B 기준)
- [ ] 회원/주문/상품/재고 데이터베이스 설계 (현재 localStorage → DB로 이전)
- [ ] 비밀번호 해시(bcrypt 등) + 세션/JWT 인증 (현재는 평문 데모)
- [ ] 관리자 페이지(주문 조회·상태변경·재고·환불)
- [ ] 주문 확인/배송 알림 이메일·알림톡 발송
